# VodDetail 搜索结果重设计

## 背景

当前 VodDetail 页面的搜索结果以统一的 `source-group-card` 形式呈现，每部电影/剧集都显示"N 个播放源"和一排来源按钮，电影和剧集在搜索结果层面几乎没有视觉区分。用户需要先点击来源，再在下方加载具体的集数/播放地址，操作路径较长。

## 目标

1. **区分电影与剧集的交互模式**：电影一步播放，剧集直接选集
2. **提升信息密度**：在卡片上展示评分、年代、类型、主演、集数状态
3. **充分利用横向空间**：左右分栏布局，避免中间留白
4. **减少操作步骤**：预加载第一个来源的 detail，卡片内即可播放/选集

## 设计

### 布局结构

采用左右分栏的紧凑列表卡片布局：

- **左侧信息区**：固定宽度（约 240px，桌面端），包含海报缩略图、标题、评分徽章、年份、类型、主演（截断）、集数状态
- **右侧操作区**：`flex: 1`，占据剩余全部空间

### 电影卡片

```
┌──────────────────────────────────────────────────────────────┐
│  ┌───┐  流浪地球                    [▶ 立即播放] [线路2] [+2] │
│  │海报│  7.9分 · 2019 · 科幻/冒险                             │
│  └───┘  吴京/屈楚萧/李光洁                                    │
└──────────────────────────────────────────────────────────────┘
```

- **「立即播放」按钮**：默认走第一个来源直接播放，高亮样式
- **备用线路按钮**：点击直接切换来源播放
- **来源过多时**：显示 `+N`，点击弹出全部来源选择面板
- **清晰度标注**：按钮下方小字标注清晰度（如 1080P），需后端支持

### 剧集/综艺/动漫卡片

```
┌──────────────────────────────────────────────────────────────┐
│  ┌───┐  三体                       [01] [02] [03] [04] [05]  │
│  │海报│  8.7分 · 2023 · 科幻/悬疑   [06] [07] [08] [09] [10] │
│  └───┘  已完结 · 30集              [11] [12] [⋯]             │
└──────────────────────────────────────────────────────────────┘
```

- **集数网格**：直接展示在右侧操作区，左对齐，自适应换行
- **已完结**：展示前 12 集（或根据容器宽度自适应）+ 「⋯」按钮
- **连载中**：展示最近 5 集 + 「⋯」，最新集高亮
- **未更新/预告集**：置灰不可点击
- **综艺/动漫**：期数用「第N期」或「第N集」格式

### 数据流

**当前流程**：
```
搜索 → 点击来源 → 加载 provider_detail → 显示 episodes
```

**新流程**：
```
搜索 → 去重得到结果列表
       → 每个结果预加载第一个来源的 provider_detail（惰性）
       → 卡片内直接展示 episodes 或播放按钮
```

**预加载策略**：
- 搜索结果返回后，对每个去重结果延迟 300ms 预加载第一个来源的 `provider_detail`
- 使用 IntersectionObserver：卡片进入视口时触发预加载
- 只预加载第一个来源（用户最常使用的），其他来源按需加载
- 预加载失败不影响卡片展示，回退到「点击加载」状态

### 组件拆分

```
src/components/detail/
├── SearchResultCard.vue          # 新：搜索结果卡片（电影/剧集统一入口）
│   ├── MovieActionPanel.vue      # 新：电影操作面板（立即播放 + 线路）
│   └── EpisodeGrid.vue           # 新：集数网格组件
├── EpisodeGroupPanel.vue         # 现有：保持不变（用于正常 catalog flow）
└── ...
```

`SearchResultCard` 根据 `item_type` 决定右侧渲染 `MovieActionPanel` 还是 `EpisodeGrid`。

### 响应式

| 断点 | 左侧宽度 | 右侧行为 |
|---|---|---|
| ≥1024px | 固定 240px | 集数网格自适应换行 |
| 768-1024px | 固定 200px | 集数按钮缩小，显示更少 |
| <768px | 全宽，上下布局 | 电影：全宽按钮；剧集：横向滚动集数 |

### 状态管理

当前搜索相关状态全部内联在 `VodDetail.vue` 中。新设计不需要引入新的 Pinia store，但需要在 `VodDetail.vue` 中新增：

- `providerDetailCache: Map<string, ProviderDetailResult>` — 缓存预加载的 provider detail，避免重复请求
- `preloadingItems: Set<string>` — 跟踪正在预加载的 item，防止重复触发

### 错误处理

- **预加载失败**：卡片不崩溃，剧集卡片显示「点击加载集数」占位按钮
- **无可用来源**：显示「暂无播放源」空状态
- **provider detail 返回空 episodes**：显示「该视频暂无可播放内容」

## API 变动

无新增后端 API。复用现有的 `provider_detail` 和 `provider_play` 命令。

可能需要后端增强（可选，非阻塞）：
- `search_all_sources` 返回结果中增加 `quality` 字段，用于标注清晰度

## 测试策略

1. **SearchResultCard 组件测试**：
   - 电影类型渲染 `MovieActionPanel`，不渲染 `EpisodeGrid`
   - 剧集类型渲染 `EpisodeGrid`，不渲染 `MovieActionPanel`
   - 集数超过阈值时显示「⋯」按钮
   - 点击「立即播放」emit 正确事件

2. **VodDetail.vue 集成测试**：
   - 搜索结果返回后触发预加载
   - 预加载结果正确缓存和展示
   - 点击集数正确导航到播放器

## 实现范围

### 在范围内
- 新建 `SearchResultCard.vue`、`MovieActionPanel.vue`、`EpisodeGrid.vue`
- 修改 `VodDetail.vue` 的搜索结果渲染逻辑（替换现有 `source-group-card`）
- 预加载第一个来源的 provider detail
- 响应式适配

### 不在范围内
- 修改正常 catalog flow 的 `EpisodeGroupPanel`（保持现状）
- 后端 API 改动（清晰度标注为可选增强）
- 播放器本身的改动
