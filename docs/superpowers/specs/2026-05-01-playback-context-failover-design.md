# 播放页上下文保留与非阻塞自动切源设计

## 背景

当前播放页有两个可见问题：

- 顶部只显示“第几集”或线路信息，没有稳定显示片名/剧名。
- 点击剧集的多源后，如果某个源解析失败，右侧抽屉会停留在遮罩式 loading 状态，剧集列表和线路列表被盖住，用户后续无法继续选择。

代码里已经存在自动 failover 和本集多源状态，但它们和页面展示耦合得过紧：

- `PlayerPage.vue` 依赖异步加载的 `detailStore.item` 来补全标题。
- `PlaybackDrawer.vue` 把 `loading` 直接当成整块骨架屏条件。
- 多源解析失败时，状态更新会继续发生，但 UI 没有把“解析中”和“已有内容可交互”区分开。

## 目标

1. 顶部标题始终优先显示片名/剧名，其次才降级为集数或线路信息。
2. 只要已经有剧集列表或线路列表，抽屉就必须保持可见，不再被 loading 整块遮住。
3. 多源失败时继续自动尝试下一个候选源或下一个播放源，直到成功或全部失败。
4. 全部失败时保留列表和失败状态，方便用户手动重试，而不是回到空白 loading。
5. 不引入新的后端接口，不改现有播放解析接口语义。

## 非目标

- 不重构 Rust 播放解析链路。
- 不为每个外部站点单独做适配。
- 不做持久化的源健康缓存。
- 不改播放器本体的控制交互，只调整上下文和失败态展示。

## 推荐方案

采用“播放上下文快照 + 非阻塞 drawer + 现有 failover 复用”的方案。

核心思路：

- 进入播放页时，把片名、当前集、剧集组、源名等信息作为上下文快照优先带入。
- 播放抽屉只在“完全没有内容可展示”时才显示骨架屏。
- 自动切源继续由现有 session failover 驱动，但失败不会清空列表。

这个方案比纯 UI 修补更稳，也比重新做一套播放会话状态层更轻。

## 架构

### 1. 上下文优先级

播放页标题和抽屉标题不再只依赖 `detailStore.item`。优先级改为：

1. 播放上下文快照中的标题
2. `detailStore.item.title`
3. route query 中的 `title`
4. 当前集标签或线路标签兜底

这样就算详情请求慢一点，页面也不会退化成“只剩第几集”。

### 2. 非阻塞抽屉

`PlaybackDrawer.vue` 里的 `loading` 只代表“数据还在补充”，不代表“必须隐藏已有内容”。

只在以下情况下显示骨架屏：

- 没有 `sources`
- 没有 `unifiedEpisodes`
- 没有 `episodeSourceAttempts`

只要其中任意一种内容已存在，就展示真实列表，并把 loading 降级成轻量状态提示。

### 3. 自动切源

多源播放继续沿用现有 `EpisodePlaybackSession` 和 failover 流程，但补齐以下约束：

- 当前候选失败后，先切同源下一个候选。
- 同源候选耗尽后，再切同集下一个播放源。
- 解析失败和视频错误都走同一条 failover 路径。
- 用户手动切源时，允许重试失败源，不受自动跳过策略影响。

### 4. 失败态展示

失败态只更新状态，不清空内容：

- 抽屉继续显示剧集列表、线路列表、本集播放源。
- 当前失败源显示失败原因或“本次失败”。
- 只有全部失败时，顶部状态才展示明确的失败文本。

## 数据流

1. 用户从详情页点击剧集或源。
2. 详情页先把当前播放上下文写入播放器 store。
3. 播放页挂载后先读取上下文快照，再读取异步详情，再读取路由参数。
4. 抽屉依据“是否已有内容”决定显示 skeleton 还是真实列表。
5. 若候选解析失败，session failover 自动推进到下一个候选或下一个源。
6. 若全部失败，页面保留列表和失败原因，允许用户手动重试。

## 错误处理

- 详情尚未到达时，不显示空标题，不隐藏已有列表。
- 解析失败时不清空当前剧集列表。
- `current源没有可用候选线路` 这类内部错误不直接作为空白页状态。
- 全部源失败时才向用户展示最终失败提示。
- 自动播放被系统拦截时，只提示用户，不把源记为不可用。

## 测试策略

### 组件测试

- `PlaybackDrawer` 在有 `unifiedEpisodes` 或 `sources` 时，不应因为 `loading` 而只渲染骨架屏。
- `PlaybackDrawer` 在系列模式下仍能显示本集播放源和失败状态。
- `PlaybackDrawer` 在失败态下仍保留剧集列表和线路列表。

### 页面测试

- `PlayerPage` 在上下文快照存在时，顶部标题优先显示片名/剧名。
- `PlayerPage` 在详情未到达时，不会退化成只有“第几集”的标题。
- `PlayerPage` 在候选失败后会自动尝试下一个候选或下一个源。
- `PlayerPage` 在全部失败时保留列表，不进入空白 loading。

### 手工验证

- 从详情页进入播放器，确认顶部标题先显示片名，再显示集数。
- 点击多源后故意触发一个失败源，确认页面继续切换到下一个可用源。
- 模拟所有源失败，确认右侧列表仍可见且可手动重试。

## Rollout Notes

- 先改展示和上下文传递，再调自动切源的失败态收口。
- 保持 `playbackSession` 逻辑不变，只补 UI 语义和上下文来源。
- 如果某些页面入口暂时拿不到上下文，允许继续使用 route query 兜底。
